# 🗺️ Vue 3 + Leaflet + ECharts Blueprint
A blueprint for a Philippine filled map visualization app using **Vue 3**, **Leaflet**, and **ECharts**.  
Supports multi-level filtering, metric overlays, and data import from CSV, Excel, JSON, and Google Sheets.

## 🧩 Component Responsibilities

### **MapView.vue**
- Renders the Philippine GeoJSON map using **Leaflet**
- Colors polygons based on the selected metric
- Responds to filters (region, province, city)
- Emits events to update the data store

### **MetricOverlay.vue**
- Displays ECharts-based visualizations (summary metrics, comparisons, charts)
- Dynamically updates based on filtered dataset

### **DataUploader.vue**
- Accepts CSV, Excel, JSON file uploads
- Parses data using **PapaParse** or **SheetJS**
- Google Sheets import via public CSV URL or API key

### **Filters.vue**
- Dropdown selectors for Region → Province → City
- Supports multi-level and simultaneous filtering

### **Legend.vue**
- Displays a color gradient scale for map metric values

## 🧠 State Management (Pinia Store)
`dataStore.js`
```js
import { defineStore } from 'pinia'

export const useDataStore = defineStore('dataStore', {
  state: () => ({
    dataset: [],
    filteredData: [],
    geoData: null,
    selectedMetric: null,
    filters: { region: null, province: null, city: null },
  }),
  actions: {
    setDataset(data) { this.dataset = data },
    setGeoData(geo) { this.geoData = geo },
    applyFilters() {
      this.filteredData = this.dataset.filter(row => {
        return (!this.filters.region || row.region === this.filters.region) &&
               (!this.filters.province || row.province === this.filters.province) &&
               (!this.filters.city || row.city === this.filters.city)
      })
    }
  }
})
```

## 🧮 Interaction Flow
1. User uploads or connects a dataset.
2. App detects geographic columns (region, province, city).
3. User selects a metric (e.g., population, revenue).
4. Leaflet renders filled polygons with color scale.
5. ECharts displays metric overlay charts.
6. Filters update both components simultaneously.
